에셋은 프로젝트에 사용할 수 있는 모든 항목을 의미
3D모델, 이미지 파일(텍스처), 머티리얼, 오디오 파일이 포함
이러한 여러 에셋을 조합하면 유니티 경험에 추가할 만한 콘텐츠 제작 가능.
에셋은 사소하지만, 함께 활용 시 몰입도 높은 환경 제작 가능.

예) 작은 집이 포함된 씬을 생성한다면,
이 집을 만들기 위해 필요한 유니티 에셋스토어에서 프로젝트로 에셋을 임포트가 필요.
1. 에셋 스토어로 이동
2. 에셋 스토어 홈페이지 둘러보기
3. 에셋 찾기
4. 에셋 다운로드

예) 건축물을 시각화 할 때에는 가구 에셋(의자, 테이블, 소파), UI요소(버튼, 토글, 이미지), 음향효과(문열림, 날씨음향, 음악)를 사용할 수 있다.

에셋에 액세스 하고 이를 유니티 프로젝트로 임포트 하는 직접 적인 방법 중 하나는 유니티 에셋스토어를 이용.
유니티 테크놀로지스와 유니티 구성원이 제작한 무료 에셋과 상용 에셋이 있는 방대한 라이브러리가 에셋 스토어.
개별 오브젝트, 머티리얼/텍스처, 애니메이션, 튜토리얼 등 다양한 에셋을 찾을 수 있음.

유니티 기본 에디터 인터페이스

scene view / game view
기본 유니티 에디터 레이아웃의 가운데는 scene view.
씬 뷰는 사용자가 제작하는 월드와 상호 작용하는 창으로, 씬 뷰를 사용해서 오브젝트를 조작하고 다양한 각도로 확인 가능.
게임 뷰를 사용해 게임의 플레이 테스트 진행

hierachy 계층창에서는 프로젝트의 모든 요소를 구성 가능. 요소 = 게임 오브젝트
씬 뷰를 통해 프로젝트에 추가한 게임 오브젝트는 계층창의 목록에 표시.
씬에서 게임 오브젝트를 삭제 시 목록에 표시되지 않음.

project 창에서는 실제 사용 여부와 관계없이 프로젝트에 사용가능한 모든 파일(에셋) 확인 가능.
프로젝트창은 탐색기처럼 폴더로 구성. 프로젝트 창에서 씬뷰로 에셋을 드래그하여바로 씬에 추가 가능.
프로젝트 창과 계층 창의 차이: 계층 창에는 현재 씬의 모든 게임 오브젝트가 포함되며, 프로젝트 창에는 프로젝트에서 전반적으로 사용할 수 있는 모든 에셋이 포함.

inspector 창에서는 게임 오브젝트에 대한 자세한 정보를 확인하고 설정 변경 가능.
씬 뷰 또는 계층 창에서 게임 오브젝트를 선택하면 사용중인 컴포넌트가 인스펙터 창에 표시.
컴포넌트는 게임 오브젝트의 프로퍼티와 동작을 설명.

툴바 
툴바는 항상 유니티 에디터 인터페잇의 상단에 위치. 
툴바 버튼을 사용해 게임 오브젝트의 선택과 조정, 씬에서의 시점 변경, 플레이 모드의 시작 및 중지 등이 가능.


유니티 에디터의 프로젝트는 씬으로 구성.
씬은 작업자가 제작하는 경험의 모든 요소를 담는 컨테이너.
씬을 별개의 경험으로 간주 가능.
예) 게임의 각 레벨이 별도의 씬이 될 수 있으며, 게임의 메인 메뉴가 또 다른 씬이 될 수 있음.
